## Output Instructions

### xsl:value-of

Evaluates an XPath expression and inserts the result as text.

```xml
<xsl:value-of select="name"/>
<xsl:value-of select="concat('$', price)"/>
<xsl:value-of select="count(//item)"/>
```

**C# parallel:** `@Model.Name` in Razor, or `<%= expression %>` in classic ASP.NET.

**Separator** — when the expression returns a sequence:
```xml
<xsl:value-of select="//tag" separator=", "/>
(: outputs "tag1, tag2, tag3" :)
```

### xsl:text

Outputs literal text with explicit control over whitespace:

```xml
<xsl:text>Hello, </xsl:text>
<xsl:value-of select="name"/>
<xsl:text>!</xsl:text>
```

Useful when you need precise whitespace control — XSLT preserves whitespace in literal text, but `xsl:text` makes it explicit.

### xsl:comment

Outputs an HTML/XML comment:

```xml
<xsl:comment>Generated by XSLT on <xsl:value-of select="current-date()"/></xsl:comment>
(: outputs: <!-- Generated by XSLT on 2026-03-19 --> :)
```

### xsl:processing-instruction

Outputs a processing instruction:

```xml
<xsl:processing-instruction name="xml-stylesheet">
  type="text/css" href="style.css"
</xsl:processing-instruction>
(: outputs: <?xml-stylesheet type="text/css" href="style.css"?> :)
```

## Control Flow

### xsl:if

Simple conditional — no else branch.

```xml
<xsl:if test="price > 50">
  <span class="badge">Premium</span>
</xsl:if>

<xsl:if test="@out-of-print">
  <span class="badge warning">Out of Print</span>
</xsl:if>

<xsl:if test="position() = last()">
  <span class="last-item"/>
</xsl:if>
```

**C# parallel:** `@if (price > 50) { ... }` in Razor.

### xsl:choose / xsl:when / xsl:otherwise

Multi-branch conditional — like C#'s `switch` or `if/else if/else`:

```xml
<xsl:choose>
  <xsl:when test="price > 100">
    <span class="tier">Premium</span>
  </xsl:when>
  <xsl:when test="price > 50">
    <span class="tier">Standard</span>
  </xsl:when>
  <xsl:otherwise>
    <span class="tier">Budget</span>
  </xsl:otherwise>
</xsl:choose>
```

**C# parallel:**
```csharp
if (price > 100) { "Premium" }
else if (price > 50) { "Standard" }
else { "Budget" }
```

`xsl:otherwise` is optional. If no `xsl:when` matches and there's no `xsl:otherwise`, nothing is output.

## Variables and Parameters

### xsl:variable

Declares a variable. **Variables are immutable** — once set, they cannot be changed. This is the biggest mental shift for imperative programmers.

```xml
<!-- Simple value -->
<xsl:variable name="tax-rate" select="0.08"/>

<!-- Computed value -->
<xsl:variable name="total" select="sum(//item/price)"/>

<!-- Complex content (the variable holds a node tree) -->
<xsl:variable name="header">
  <h1><xsl:value-of select="$site-title"/></h1>
  <p>Welcome to our site.</p>
</xsl:variable>

<!-- Use the variable -->
<p>Total with tax: $<xsl:value-of select="$total * (1 + $tax-rate)"/></p>
<xsl:copy-of select="$header"/>
```

**Why immutable?** XSLT is a functional language. Immutability enables the engine to parallelize, optimize, and stream transformations. If you need to "update" a variable, use recursion or accumulator patterns.

**Scope:** Variables are scoped to their containing element. A variable declared inside `xsl:for-each` is local to each iteration.

### xsl:param

Declares a parameter — like a variable but can be set by the caller.

**Stylesheet parameters** (set from outside — CLI, API):
```xml
<xsl:stylesheet ...>
  <xsl:param name="base-url" select="'/'"/>
  <xsl:param name="site-title" select="'My Site'"/>
```

**Template parameters** (set from calling templates):
```xml
<xsl:template match="product">
  <xsl:param name="show-price" select="true()"/>
  ...
</xsl:template>
```

## Iteration and Sorting

### xsl:for-each

Iterates over a sequence. Inside the loop, `.` is the current item.

```xml
<table>
  <xsl:for-each select="//product">
    <tr>
      <td><xsl:value-of select="name"/></td>
      <td><xsl:value-of select="price"/></td>
    </tr>
  </xsl:for-each>
</table>
```

**When to use `xsl:for-each` vs `xsl:apply-templates`:**
- `xsl:for-each` — when the rendering logic is simple and specific to one location
- `xsl:apply-templates` — when different node types need different rendering, or when you want reusable templates

### xsl:sort

Sorts the items processed by `xsl:for-each` or `xsl:apply-templates`:

```xml
<xsl:for-each select="//product">
  <xsl:sort select="price" data-type="number" order="ascending"/>
  <xsl:sort select="name" order="ascending"/>  <!-- secondary sort -->
  ...
</xsl:for-each>

<xsl:apply-templates select="//product">
  <xsl:sort select="@category"/>
  <xsl:sort select="name"/>
</xsl:apply-templates>
```

**Attributes:**
| Attribute | Values | Default |
|-----------|--------|---------|
| `select` | XPath expression for sort key | `.` (string value) |
| `order` | `ascending`, `descending` | `ascending` |
| `data-type` | `text`, `number` | `text` |
| `collation` | Collation URI | Default collation |

### xsl:for-each-group

Groups items and iterates over the groups. One of XSLT's most powerful features.

```xml
<!-- Group products by category -->
<xsl:for-each-group select="//product" group-by="@category">
  <section>
    <h2><xsl:value-of select="current-grouping-key()"/></h2>
    <ul>
      <xsl:for-each select="current-group()">
        <li><xsl:value-of select="name"/></li>
      </xsl:for-each>
    </ul>
  </section>
</xsl:for-each-group>
```

**Output:**
```html
<section>
  <h2>electronics</h2>
  <ul>
    <li>Wireless Mouse</li>
    <li>Mechanical Keyboard</li>
  </ul>
</section>
<section>
  <h2>accessories</h2>
  <ul>
    <li>USB-C Hub</li>
  </ul>
</section>
```

**C# parallel:** `items.GroupBy(p => p.Category)`

**Grouping modes:**

| Attribute | Groups by... |
|-----------|-------------|
| `group-by` | Key value (like `GroupBy`) |
| `group-adjacent` | Adjacent items with same key (like `ChunkBy`) |
| `group-starting-with` | Pattern match starts a new group |
| `group-ending-with` | Pattern match ends a group |

## Copying

### xsl:copy

Shallow copy — copies the current node without its children:

```xml
<xsl:template match="*">
  <xsl:copy>
    <xsl:apply-templates select="@* | node()"/>
  </xsl:copy>
</xsl:template>
```

This is the **identity transform** — the most important XSLT pattern. It copies the entire document unchanged. You then override specific templates to modify specific elements.

### xsl:copy-of

Deep copy — copies a node and all its descendants:

```xml
<xsl:copy-of select="//product[@id='P001']"/>  <!-- copies the entire product subtree -->
<xsl:copy-of select="$variable"/>              <!-- copies a variable's content -->
```

### The Identity Transform Pattern

The most common XSLT pattern: copy everything, then override what you need to change.

```xml
<!-- Copy everything by default -->
<xsl:mode on-no-match="shallow-copy"/>

<!-- Override just the price to add currency symbol -->
<xsl:template match="price">
  <price>$<xsl:value-of select="."/></price>
</xsl:template>
```

The XSLT 3.0 `on-no-match="shallow-copy"` declaration replaces the verbose identity template from XSLT 1.0. It means: "for any node without an explicit template, copy it and process its children."

## Dynamic Construction

### xsl:element

Creates an element with a computed name:

```xml
<xsl:element name="h{@level}">
  <xsl:apply-templates/>
</xsl:element>
(: Creates <h1>, <h2>, etc. based on the level attribute :)
```

### xsl:attribute

Creates an attribute dynamically:

```xml
<div>
  <xsl:attribute name="class">
    <xsl:text>product</xsl:text>
    <xsl:if test="@featured"> featured</xsl:if>
  </xsl:attribute>
</div>
(: outputs <div class="product featured"> or <div class="product"> :)
```

**Tip:** For simple cases, use attribute value templates instead:
```xml
<div class="product {if (@featured) then 'featured' else ''}"/>
```

## Grouping

### position() and last()

Inside `xsl:for-each` or template matching, these functions return the current position and total count:

```xml
<xsl:for-each select="//product">
  <div class="product{if (position() = last()) then ' last' else ''}">
    <span class="number"><xsl:value-of select="position()"/></span>
    <xsl:value-of select="name"/>
  </div>
</xsl:for-each>
```

### xsl:number

Generates formatted numbers for numbering schemes:

```xml
<xsl:number format="1. "/>     (: 1. 2. 3. :)
<xsl:number format="a) "/>     (: a) b) c) :)
<xsl:number format="I. "/>     (: I. II. III. :)
<xsl:number level="multiple" count="section" format="1.1 "/>
(: 1.1 1.2 2.1 2.2 — multi-level numbering :)
```

## Multiple Output Documents

### xsl:result-document

Writes output to a different file. Essential for generating multi-page sites.

```xml
<xsl:template match="product">
  <!-- Write each product to its own HTML file -->
  <xsl:result-document href="products/{@id}.html">
    <html>
      <head><title><xsl:value-of select="name"/></title></head>
      <body>
        <h1><xsl:value-of select="name"/></h1>
        <p>$<xsl:value-of select="price"/></p>
      </body>
    </html>
  </xsl:result-document>
</xsl:template>
```

This is how Crucible generates one HTML page per documentation page from a single XSLT transformation.

## User-Defined Functions

### xsl:function

Define reusable functions callable from XPath expressions:

```xml
<xsl:function name="my:format-price" as="xs:string"
              xmlns:my="http://example.com/my">
  <xsl:param name="amount" as="xs:decimal"/>
  <xsl:param name="currency" as="xs:string" select="'USD'"/>
  <xsl:sequence select="
    if ($currency = 'USD') then concat('$', format-number($amount, '#,##0.00'))
    else if ($currency = 'EUR') then concat('€', format-number($amount, '#,##0.00'))
    else concat($amount, ' ', $currency)
  "/>
</xsl:function>

<!-- Use it anywhere in XPath: -->
<span class="price"><xsl:value-of select="my:format-price(price, price/@currency)"/></span>
```

**C# parallel:** A static utility method:
```csharp
static string FormatPrice(decimal amount, string currency = "USD") => ...
```

**Key advantage:** Custom XSLT functions are callable from XPath expressions — including predicates, sort keys, and attribute value templates. They extend XPath's function library within your stylesheet.
